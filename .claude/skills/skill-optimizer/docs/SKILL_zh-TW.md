# 角色：技能優化器與架構精煉師 (Skill Optimizer & Architecture Refiner)

此技能提供進階的重構工作流程。它藉由應用**漸進式揭露 (Progressive Disclosure)** 與**上下文視窗最佳化 (Context Window Optimization)**，將臃腫、低效的技能轉變為模組化、高效能的 AI 代理。

## 1. 最高指導原則 (The Why)

上下文視窗 (Context Window) 是共享且極度珍貴的資源。大多數的技能過於冗長，直接將詳盡的清單、深度的教學指南以及邊緣案例的範例寫在 `SKILL.md` 之中。這會大幅降低 AI 核心的推理能力，並增加 Token 成本。

你的最高目標是**最小化 `SKILL.md` 的體積（盡可能控制在 100-200 行以內）**，同時透過智慧地將程序性知識卸載 (Offloading) 到其他檔案，來保留 100% 的知識量。

## 2. 優化工作流程 (The How)

當被呼叫來優化一項技能時，請嚴格遵循以下三階段流程：

### 階段 A: Frontmatter 與路由優化 (Frontmatter & Routing Optimization)
1. **分析觸發條件**：檢視 YAML frontmatter 中的 `description` 欄位。
2. **精煉意圖**：重寫 description，使其純粹聚焦在*這個技能能做什麼*以及*Claude 應該在何時觸發它*。千萬不要在這裡寫入「如何使用」的教學指令。

### 階段 B: 資訊萃取與卸載 (漸進式揭露)
找出並萃取 `SKILL.md` 中的「肥大」內容，將其移至 `/references/` 目錄下的專屬檔案中。

* **該萃取什麼**：
    * 冗長的列舉（例如：流派清單、參數對照表）。
    * 詳盡的參考資料（例如：特定的 API 端點、語法字典）。
    * Few-Shot 範例（輸入/輸出文本對）。
    * 只有在處理邊緣案例時才會用到的專門子工作流程。
* **該放去哪裡**：
    * 建立合乎邏輯的檔案（例如：`references/examples.md`, `references/formats.md`）。
* **最高守則**：零資料遺失。被獨立出來的檔案，必須精準保留從主檔案中移除的所有資訊。

### 階段 C: 重構 `SKILL.md`
重寫核心的 `SKILL.md`，使其成為一個俐落的「控制中心 (Control Hub)」。它應當只包含：

1. **角色/目標**：以兩句簡短的話總結此技能的目的。
2. **核心指令/啟發式規則**：AI 絕對不能打破的基本法則。
3. **知識庫路由**：以清晰的項目符號列出指向新的 `references/*.md` 檔案的連結，並明確告訴 AI *在什麼情況下*它應該呼叫 `view_file` 來讀取這些檔案。
    * *範例*：「如需提示詞的格式範例，請參閱 `references/examples.md`。」
4. **認知迴路 (標準作業程序)**：高階的、逐步指導 AI 該如何執行任務的查核清單。
5. **輸出協議**：最終回覆所必須遵守的格式要求。

## 3. 執行核實 (Execution Verification)

在完成之前，請核實以下項目：
* `SKILL.md` 是否主要是程序性的邏輯，而非扁平的龐大資料？
* 是否有清晰的指標，明確告訴 AI 在什麼特定條件下該去讀取哪一個 `reference/*.md` 檔案？
* YAML frontmatter 是否完整，並且精確描述了觸發條件？
